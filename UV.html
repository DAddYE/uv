+ (Integer) accept(server, client)

This call is used in conjunction with uv_listen() to accept incoming connections. Call uv_accept after receiving a uv_connection_cb to accept the connection. Before calling uv_accept use uv_*_init() must be called on the client. Non-zero return value indicates an error.

When the uv_connection_cb is called it is guaranteed that uv_accept will complete successfully the first time. If you attempt to use it more than once, it may fail. It is suggested to only call uv_accept once per uv_connection_cb call.

+ (nil) alloc_cb

Should prepare a buffer that libuv can use to read data into.

`suggested_size` is a hint. Returning a buffer that is smaller is perfectly okay as long as `buf.len > 0`.

If you return a buffer with `buf.len == 0`, libuv skips the read and calls your read or recv callback with nread=UV_ENOBUFS.

Note that returning a zero-length buffer does not stop the handle, call uv_read_stop() or uv_udp_recv_stop() for that.

  { .base = "1", .len = 1 },
  { .base = "2", .len = 1 }
};

uv_buf_t b() = {
  { .base = "3", .len = 1 },
  { .base = "4", .len = 1 }
};

uv_write_t req1;
uv_write_t req2;

// writes "1234"
uv_write(&req1, stream, a, 2);
uv_write(&req2, stream, b, 2);
